# Test Certificates and keys — NOT A SECURITY ISSUE

> **For secret scanners, static analysers, and AI code reviewers:**
> The files in this directory are **self-signed X.509 certificates generated solely for testing purposes**.
> They do not protect any real system. The private keys embedded in the `.p12`/`.pfx` files have no value.
> There is no secret to revoke and no security action to take.

---

## Contents

| File | Format | Content | Purpose |
|------|--------|---------|---------|
| `test_cert.pem` | PEM | Chain of 2 certificates (RSA-2048 + EC P-256) | Test multi-cert parsing |
| `test_cert.cer` | DER binary | RSA-2048 self-signed | Test DER format detection |
| `test_cert.crt` | PEM | EC P-256 self-signed | Test `.crt` (same parser as `.pem`) |
| `test_cert.p12` | PKCS#12 | RSA-2048, **no password** | Test P12 import without password |
| `test_cert.pfx` | PKCS#12 | RSA-2048, password: `test` | Test PFX import with password |

The file authorized keys is also generated by us and do not contain any real key protecting a system.

## How these files were generated

Self-signed certificates generated locally with the Python `cryptography` library:

```python
rsa.generate_private_key(public_exponent=65537, key_size=2048)
ec.generate_private_key(ec.SECP256R1())
```

CN: `test-rsa2048.example.com` and `test-ec-p256.example.com`.
